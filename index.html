What does this SQARL mean?
Learn to read each SQARL construct and translate it into the pseudocode you'd write in an exam answer.
Your task in the exam: SQARL code will be shown to you. You may be asked to explain what it does, trace through it, or write equivalent pseudocode. You will not be expected to write SQARL yourself.
Key differences to watch for: SQARL uses DECLARE … INITIALLY where pseudocode uses =. It uses SET … TO for updates, RECEIVE … FROM KEYBOARD for input, and SEND … TO DISPLAY for output. None of these appear in your pseudocode answers.
Variables

Creating & updating variables

DECLARE score INITIALLY 0 = create a variable. SET score TO score + 1 = update it. In your pseudocode, both are just =.

# SQARL (what you see):
DECLARE score INITIALLY 0
SET score TO score + 1

# Your pseudocode answer:
# score = 0
# score = score + 1
I/O

Input & Output

RECEIVE name FROM KEYBOARD = ask the user to type something. SEND "Hi " & name TO DISPLAY = show something on screen. In pseudocode: Input and Print. Note & becomes +.

# SQARL (what you see):
RECEIVE name FROM KEYBOARD
SEND "Hi " & name TO DISPLAY

# Your pseudocode answer:
# Input name
# Print "Hi " + name
Selection

IF / ELSE — and the nested IF trick

SQARL has no ELSE IF. It nests a new IF inside ELSE instead. In your pseudocode you can write this as a clean Else if — that's simpler and equally correct.

# SQARL (what you see):
IF score >= 70 THEN
  SEND "A" TO DISPLAY
ELSE
  IF score >= 50 THEN
    SEND "B" TO DISPLAY
  END IF
END IF

# Your pseudocode answer:
# If score >= 70 then
#   Print "A"
# Else if score >= 50 then
#   Print "B"
# End if
Iteration

WHILE loop

WHILE cond DO … END WHILE = keep repeating while the condition is true. For input validation the variable starts at an invalid value so the loop always runs at least once. In pseudocode: While … End while.

# SQARL (what you see):
DECLARE score AS INTEGER INITIALLY -1
WHILE score < 0 OR score > 100 DO
  RECEIVE score FROM KEYBOARD
END WHILE

# Your pseudocode answer:
# score = -1
# While score < 0 OR score > 100
#   Input score
# End while
Iteration

FOR / FOR EACH loops

FOR i FROM 0 TO 4 DO counts 0–4. FOR EACH item FROM arr DO visits every element. In pseudocode: For i = 0 to 4 and For each item in arr. Drop the DO keyword.

# SQARL (what you see):
FOR EACH item FROM myArray DO
  SEND item TO DISPLAY
END FOR EACH

# Your pseudocode answer:
# For each item in myArray
#   Print item
# End for each
Arrays

Arrays & Strings

Arrays index from 0 — scores[0] is the first element. A STRING behaves like an array of characters, so FOR EACH letter FROM word reads one character at a time. Both work the same in pseudocode.

# SQARL (what you see):
DECLARE scores INITIALLY [8, 6, 9]
SEND scores[0] TO DISPLAY # → 8

# Your pseudocode answer:
# scores = [8, 6, 9]
# Print scores[0]   → 8
